## JWT JWK 埋め込み攻撃（JWT authentication bypass via JWK header injection）

### 1. どんな脆弱性？
サーバーが **「JWT の署名検証に使う公開鍵」** を  
**JWT ヘッダーの `jwk` に入っている公開鍵をそのまま信じて使ってしまう** 脆弱性。

本来：

```
署名検証に使う公開鍵はサーバーが固定で持っているべき
```

脆弱なサーバー：

```
ユーザーがJWTに入れてきた公開鍵をそのまま採用してしまう
```

その結果：

```
攻撃者が自分で作った秘密鍵で署名したJWTを
攻撃者が自分で入れた公開鍵でサーバーが検証してしまう
→ 署名が常に正しい扱いになる
```

---

## 2. どう気づく？
Burp で JWT を見ると、ヘッダーに **`jwk` を受け入れる実装**がある。  
また、署名検証に失敗してもエラーが出ず、`jwk` を変えても動作が変わる。

気づきポイント：

- JWT ヘッダーに `jwk` を入れてもサーバーが拒否しない  
- `kid` や `jwk` を変えても署名検証の挙動が変わる  
- 公開鍵を外部から受け入れている可能性が高い

---

## 3. どう攻撃する？
攻撃の本質は **「自分の鍵ペアで署名した JWT をサーバーに信じさせる」**。

手順はこう：

1. **自分で RSA キーペアを生成**（Burp JWT Editor Keys）
2. JWT の payload を改ざん  
   例：`"sub": "administrator"`
3. 自分の秘密鍵で署名し直す
4. JWT ヘッダーに **自分の公開鍵を `jwk` として埋め込む**
5. サーバーがその公開鍵で署名を検証 → **成功してしまう**
6. 管理者として `/admin` にアクセス可能

---

## 4. どう対策する？
対策は明確で、実務でもよく問われる。

- **署名検証に使う公開鍵はサーバー側で固定し、外部入力を絶対に信用しない**
- JWT ヘッダーの `jwk` や `kid` を **外部から受け入れない**
- ライブラリの **JWK auto-resolve 機能を無効化**
- 署名アルゴリズムを強制（`alg` をクライアントに選ばせない）
- 署名検証前にヘッダーを厳格にバリデーション

---

## 5. 面接用 30秒まとめ（暗記用）

> **「JWT の署名検証に使う公開鍵を、JWT ヘッダーの `jwk` から動的に読み込んでしまう脆弱性です。攻撃者は自分で鍵ペアを作り、自分の秘密鍵で署名した JWT を、自分の公開鍵を `jwk` に埋め込んで送ることで、サーバーに正しい署名だと誤認させられます。結果として任意のユーザーに成りすまし可能です。対策は、署名検証に使う公開鍵をサーバー側で固定し、`jwk` や `kid` を外部入力から受け入れないことです。」**

---

🔥 結論：公開鍵は「署名が本物かどうかを確認するための鍵」

もっと噛み砕くとこう。

• 秘密鍵（private key）
　→ サーバーだけが持ってる
　→ JWT の署名を「作る」ための鍵
• 公開鍵（public key）
　→ 誰に見られてもOK
　→ JWT の署名が「本物か確認する」ための鍵


つまり、

👉 秘密鍵で署名を作り、公開鍵で署名を検証する

これが RS256（公開鍵暗号方式）の仕組み。

---

🍎 図で見ると一瞬で理解できる

JWT = header.payload.signature

signature = 署名(header + payload, 秘密鍵)

検証 = 検証(signature, 公開鍵)


• 秘密鍵 → 作る側
• 公開鍵 → 確認する側


---

🔥 じゃあなんで公開鍵が必要なの？

理由はシンプル。

✔ サーバーが「この JWT は本物か？」を確認するため

✔ クライアントは署名を作れないようにするため

もし秘密鍵をクライアントに渡したら、
攻撃者が自由に JWT を偽造できてしまう。

だから、

• 秘密鍵はサーバーだけが持つ
• 公開鍵は誰でも見れる


という構造になってる。

---

🧩 HS256 と RS256 の違い（ここが超重要）

あなたが混乱してたのはここ。

✔ HS256（共通鍵方式）

• 秘密鍵 = 署名作成にも検証にも使う
• 同じ鍵を両方で使う
• → 鍵が漏れたら終わり
• → hashcat で総当たりされるのはこっち


✔ RS256（公開鍵方式）

• 秘密鍵 → 署名作成
• 公開鍵 → 署名検証
• → 公開鍵が漏れても問題なし
• → 秘密鍵は絶対に漏れないようにする


---

🔥 つまり、あなたの質問に答えるとこう

公開鍵は何なん？

→ 署名が本物かどうかを確認するための鍵。
　秘密鍵で作った署名を、公開鍵で検証する。

秘密鍵を総当たりしてるってどういうこと？

→ 総当たりされるのは HS256 のときだけ。
　RS256 では総当たりできない（仕組みが違う）。
