
# 🔥 フェーズ1：UNION が「通る形」かどうかを確認（ここが肝）

## 🎯 やること  
Repeater で **category の値だけ**を次の形に変える  
（※ラボ本文に書いてあるやつ）

```
'+UNION+SELECT+'abc','def'#
```

---

# 🧭 じゃあ「UNION が通った」ってどう判断する？

判断基準はこの **3つだけ**。

---

## ✅ 判断①：エラーが出ない

- **500 が出ない**
- ステータスが **200 OK**
- 真っ白画面じゃない

👉 **SQL構文として成立している**

---

## ✅ 判断②：画面の内容が変わる

例：

- 商品一覧の中に `abc` や `def` が混ざる  
- カテゴリとは無関係な文字が表示される

👉 **UNION の結果が画面に出ている**

＝ **UNION が通っている**

---

## ✅ 判断③：元の表示＋α になっている

- 元の商品一覧はそのまま  
- その中に **自分が入れた文字列が追加される**

👉 **UNION 成功の典型パターン**

---

# ❌ 逆に「UNION が通ってない」状態は？

## ❌ パターン1：500エラー

- SQL が壊れてる  
- クォート or コメントが合ってない  

👉 **UNION 以前の問題**

---

## ❌ パターン2：画面が元と全く同じ

- エラーは出ない  
- でも `abc` も `def` も見えない  

👉  
- UNION が無視されてる  
- 表示列じゃない  
- そもそも UNION 不可  

---

# 🎯 一言でまとめると

> **「自分が入れた文字が画面に出たら UNION は通ってる」**

これが一番確実。

---

# 🔥 フェーズ2：列数と型が分かる（UNION が通った証拠）

```
'+UNION+SELECT+'abc','def'#
```

これが表示された時点で：

- 列数：**2列**
- 型：**両方テキスト**
- UNION：**成功**

👉 **ORDER BY 調査はもう不要**

---

# 🎯 フェーズ3：DBバージョン表示（ゴール）

```
'+UNION+SELECT+@@version,+NULL#
```

- バージョン文字列が画面に出る  
  👉 **ラボクリア**
